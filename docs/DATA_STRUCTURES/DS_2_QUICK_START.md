## The Efficiency Model: Why Individual + Group Saves Time

**Without This Structure:**
- Groups start discussion without prep → 30 min wasted defining roles
- Don't know who owns which code → duplicate coding/testing
- Integration bugs discovered too late → refactoring crisis
- Capstone rushed at end → poor documentation

**With This Structure:**
- Individuals prep 30 min before meeting → everyone knows their DS
- Group meeting 15-20 min → verify integration, update capstone
- Individual attribution clear → capstone writes itself
- Submit complete, verified work → ready for next sprint

**Time Savings:** 1-2 hours per group by front-loading individual work

## Key Success Indicators

You've succeeded at DS #2 if:

✓ **Individual Level:**
- Each person can explain their DS + FRQ connection clearly

- Capstone narrative is compelling + specific

- Individual + group work complements (not duplicates)

- Each person highlighted for individual contribution
- System presented as unified (not disconnected parts)

✓ **Capstone Readiness:**
- Section ready to include in final capstone document
- CS113 competency clearly demonstrated
