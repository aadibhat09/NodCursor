# Why We Started NodCursor

NodCursor began from a simple gap: everyday computing still assumes reliable hand input.

## Problem

Many users with temporary or permanent mobility constraints are forced to choose between:

- expensive specialized hardware,
- limited accessibility tooling,
- or workflows that are hard to personalize.

## Goal

Build a browser-first, open-source interaction layer that works with a webcam and no proprietary devices.

## Principles

- Local-first processing (privacy by default)
- Fast personalization via settings and presets
- Redundant input paths (head, blink, mouth, dwell, voice)
- Progressive learning curve for users and caregivers

## What Success Looks Like

- A user can complete core browser actions without hand input.
- Setup is possible in minutes, not days.
- Tuning can adapt to different fatigue levels and movement ranges.
- The project remains auditable and community-improvable.
